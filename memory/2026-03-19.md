### Architecture Decisions

**Canonical Dogfood Path:**
```
Jira Issue (ZB-022)
   ↓
[Jira Connector] Fetch issue
   ↓
[Ingestion Service] Convert to BrainTask CR (ZB-023)
   ↓
[BrainTask CR] Pending phase
   ↓
[Foreman Reconciler] Schedule task
   ↓
[Worker Pool] Dispatch to worker
   ↓
[FactoryTaskRunner] Execute task
   ↓
[Factory + LLMGenerator] Use policy YAML for provider/model (ZB-024)
   ↓
[Result Capture] Capture outcome
   ↓
[BrainTask CR] Update status (Completed/Failed)
   ↓
[Jira Connector] Update Jira issue with result (ZB-025)
   ↓
Jira Issue (resolved/commented)
```

**Key Constraints Enforced:**
1. Local Ollama must use qwen3.5:0.8b only (CPU-only inference)
2. Policy YAML drives provider/model selection (not hardcoded)
3. Fail-closed on invalid policy (no silent drift)
4. Idempotent BrainTask creation (no duplicate work)
5. Session affinity for task routing (same session → same worker)
6. Bounded safe tasks only (docs, runbooks, config, bounded refactor)
